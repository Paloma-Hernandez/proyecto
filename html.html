ORIGEN

Si bien el origen de HTML se remonta a 1991, el año 1995 también tiene especial relevancia. Esto se debe a que en 1995 se publicó la segunda versión o HTML 2.0, que en realidad era una petición de estandarización. Es decir, HTML 2.0 fue la primera propuesta oficial de estandarización de HTML.

Dos años después, en 1997, se generaron dos nuevas versiones: a principios de año se publicó la versión HTML 3.2 y, a finales del mismo año, se publicó la actualización HTML 4.0. Como puedes ver, los cambios fueron a toda velocidad y los desarrolladores web de la época estuvieron algo ocupados adaptándose a cada modificación. Esto por no hablar de las pugnas entre los fabricantes de navegadores que realizaban actualizaciones muy personalizadas, hasta el punto de que una página web podía ser incompatible de un sitio a otro.

Posteriormente, a finales del año 1999, se publicó una nueva especificación oficial: la versión HTML 4.01, que marcó otro hito para la evolución de HTML.

Hasta 2008 no recibimos una nueva entrega, el borrador de la versión HTML 5 que es, en realidad, el sustento de todas la versiones actuales.

Desde el año 2014, HTML 5 es estándar y, aunque siguen surgiendo nuevas modificaciones y actualizaciones, los cambios son mínimos y fáciles de asimilar. Por ejemplo, una de las últimas recomendaciones es HTML 5.3, que salió a principios del 2021, pero no significó un cambio traumático.

¿QUE SIGNIFICA HTML?

HTML en español se traduce como Lenguaje Marcado de Texto, es decir, un lenguaje con marcas que va a permitir estructurar y organizar el texto para que este sea legible. Además, permite, por medio de las etiquetas, establecer ciertas relaciones, hipervínculos y otras modificaciones.

CARACTERISTICAS


  • Es fácil de usar y entender.

  • Es utilizado para crear páginas web.

  • Permite describir hipertextos.

  • Permite que el usuario se mueva por cualquier sitio de Internet haciendo clic en un texto específico —hipervínculo—. Y lo que etiqueta al HTML es el marcador del sitio web, estableciendo un texto pasado dentro de un nuevo texto.

  • Está fundamentado por una serie de breves códigos escritos en un registro de texto por el desarrollador del sitio web.

  • Es multiplataforma, por lo que se puede acceder desde cualquier lugar y dispositivo.

  • No es necesario estar en línea para que el lenguaje HTML funcione correctamente, ya que se puede codificar una página web sin conexión alguna, es decir, es posible almacenar todo un sitio web en una computadora y luego transferir los registros a la web.

  • Todos los elementos de un documento HTML constan de una etiqueta de inicio, un bloque de texto y una etiqueta de cierre.

  • Tiene un despliegue rápido.

  • Es reconocido y admitido por cualquier tipo de explorador web.

    • APLICACIONES Y USOS


  • Desarrollo de páginas web: HTML se usa mucho para crear páginas que se muestran en la red mundial. Cada página contiene un conjunto de etiquetas HTML que incluyen hipervínculos que se utilizan para conectarse a otras páginas. Cada página que presenciamos, en la red mundial, está escrita con una versión de código HTML.

  • Creación de documentos web: La creación de documentos en Internet está dominada por HTML y su concepto básico a través de etiqueta y DOM, es decir, modelo de objeto de documento. Las etiquetas HTML se insertan antes y después o frases para ubicar su formato y ubicación en la página. Un documento web consta de tres secciones: título, encabezado y cuerpo. Head incluye la información para identificar el documento, incluido el título y cualquier otra palabra clave importante. Se puede ver un título en la barra del navegador y la sección del cuerpo es la parte principal del sitio web visible para el espectador. Los tres segmentos están diseñados y creados mediante el uso de etiquetas HTML. Cada sección tiene su propio conjunto específico de etiquetas, que se representan de manera dedicada manteniendo los conceptos de encabezado, título y cuerpo en un bucle.

  • Navegación por Internet: Este es uno de los usos más importantes de HTML que es revolucionario. Esta navegación es posible utilizando el concepto de hipertexto. Básicamente es un texto que se refiere a otras páginas web o texto y cuando el usuario hace clic en él, navega al texto o página referenciado. HTML se utiliza mucho para incrustar el hipervínculo dentro de las páginas web. Un usuario puede navegar fácilmente dentro de las páginas web y también entre sitios web, que se encuentran en diferentes servidores.

  • Almacenamiento del lado del cliente: Anteriormente, un usuario no podía guardar los datos del navegador del usuario que persistirían en las sesiones. Para cumplir con este requisito, se debe construir una infraestructura del lado del servidor o utilizar las cookies del usuario. Pero con HTML5, el almacenamiento del lado del cliente es factible utilizando localStorage e IndexDB. Estas dos estrategias tienen sus propios estándares y características. localStorage básicamente proporciona almacenamiento de tablas hash basadas en cadenas. Su API es muy simple y proporciona al desarrollador los métodos setItem, getItem y removeItem. IndexDB, por otro lado, es un almacén de datos más grande y mejor del lado del cliente. La base de datos IndexDB se puede ampliar con el permiso del usuario.

  • Uso de capacidades sin conexión: Una vez que los datos se pueden almacenar en el navegador, el desarrollador puede pensar en una estrategia para hacer que la aplicación funcione, cuando un usuario está desconectado. HTML5 tiene su mecanismo de caché de aplicaciones que definiría cómo el navegador gestiona la situación fuera de línea. La memoria caché de aplicaciones, responsable de la capacidad fuera de línea, en realidad se compone de diferentes componentes, que incluye métodos API que crean una actualización, leen archivos de manifiesto y eventos. Al usar cierta propiedad en HTML5, un desarrollador puede verificar si la aplicación está en línea o no. Un desarrollador también puede especificar en el archivo de manifiesto de la memoria caché de la aplicación del sitio web la información, por ejemplo, qué navegador administra los recursos para su uso sin conexión. En el archivo de manifiesto, también se pueden especificar los recursos que están disponibles sin conexión.

  • Soporte de entrada de datos con HTML: El estándar HTML5 y un conjunto de API se pueden usar para admitir el nivel de trabajo de entrada de datos. A medida que los navegadores implementan nuevos estándares HTML5, los desarrolladores simplemente pueden agregar los atributos a la etiqueta que indican los campos, el texto, el formato de datos, etc. HTML5 ha creado varios atributos nuevos para controlar los teclados en pantalla, la validación y otras experiencias de ingreso de datos. para que el usuario final pueda tener una mejor entrada de datos.

  • Uso del desarrollo del juego: Antes del advenimiento de HTML5, el desarrollo del juego era un dominio exclusivo de Flash y Silverlight. Dado que los navegadores admiten nuevas especificaciones para HTML5, incluido CSS3 y un motor JavaScript ligero para impulsar una nueva experiencia rica, HTML5 puede brindar la realidad del desarrollo de juegos posible, que antes era el fuerte de Flash y Silverlight. No es necesario implementar todas las características de las API, pero se pueden utilizar las más apropiadas y eliminar el resto de las características.

    • VERSIONES DE HTML


    Los estándares oficiales HTML son el HTML 2.0, el HTML 3.2, el HTML 4.0, el HTML 4.01 y el HTML 5. El HTLM 5 es la última especificación oficial y se espera que continúe evolucionando a lo largo de los próximos años.


  • HTML 2.0: En 1995 se publica el estándar HTML 2.0. A pesar de su nombre, HTML 2.0 es el primer estándar oficial de HTML, es decir, el HTML 1.0 no existió como estándar. HTML 2.0 no soportaba tablas. Se simplificaba al máximo la estructura del documento para agilizar su edición, donde la declaración explícita de los elementos body, html y head es opcional.

  • HTML 3.2: La versión HTML 3.2 se publicó en 1997 y es la primera recomendación de HTML publicada por el W3C (Consorcio internacional). Esta revisión incorporó los últimos avances de las páginas web desarrolladas hasta 1996, como applets de Java y texto que fluye alrededor de las imágenes.

  • HTML 4.01: La última especificación oficial de HTML se publicó en diciembre de 1999 y se denomina HTML 4.01. Desde la publicación de HTML 4.01, el W3C se centró en el desarrollo del estándar XHTML. Por este motivo, en el año 2004, las empresas Apple, Mozilla y Opera mostraron su preocupación por la falta de interés del W3C en HTML y decidieron organizarse en una nueva asociación llamada WHATWG (Web Hypertext Application Technology Working Group) que comenzó el desarrollo del HTML 5, cuyo primer borrador oficial se publicó en enero de 2008. Debido a la fuerza de las empresas que forman el grupo WHATWG y a la publicación de los borradores de HTML 5.0, en marzo de 2007 el W3C decidió retomar la actividad estandarizadora de HTML, dentro del cual decidió integrar el XHTML.

  • HTML 5, HTML 5.1, HTML 5.2: El consorcio internacional W3C, después de una evolución de varios años, liberó el HTML 5 como estándar oficial a finales de octubre de 2014. HTML 5 incorpora nuevos elementos no contemplandos en HTML 4.01. Hay diversos cambios respecto a HTML 4.01. Hay nuevas etiquetas, se introduce la posibilidad de introducir audio y video de forma directa en la web sin necesidad de plugins o complementos en los navegadores, y otras novedades. El W3C irá lanzando progresivamente nuevas evoluciones del HTML 5.
